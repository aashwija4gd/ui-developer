# Accessibility

## WCAG principles

- Perceivable.
- Operable.
- Understandable.
- Robust.

## Visual accessibility

- Body text needs strong contrast.
- Never use color as the only signal.
- Body copy should not drop below 16px.
- Layouts must remain usable at 200% zoom.
- Provide reduced-motion alternatives.

## Motor accessibility

- Touch targets must be at least 44x44px.
- Every interactive element must be keyboard accessible.
- Tab order must match the visual order.
- Focus must be visible.
- Warn before timeouts.

## Screen reader accessibility

- Every meaningful image needs alt text.
- Icon-only buttons need labels.
- Use semantic HTML.
- Every input needs a label.
- Announce errors and dynamic updates.

## Cognitive accessibility

- Keep navigation stable.
- Break complex tasks into clear steps.
- Prevent errors where possible.
- Offer undo when feasible.
- Keep language simple and explicit.
