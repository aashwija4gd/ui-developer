# Navigation

## Patterns by platform

- Mobile: bottom tab bar for primary navigation, hamburger or drawer for secondary.
- Tablet: bottom bar or side rail.
- Desktop web: left sidebar or top nav.
- Desktop app: left sidebar plus contextual panels.

## Rules

- Keep mobile tab bars to 4-5 items max.
- Prefer one-word labels, two words maximum.
- Active state must always be obvious.
- Navigation order should reflect frequency of use.
- Put destructive or admin items at the bottom.

## Back navigation

- Every drill-down screen needs a visible back path.
- On mobile, back buttons belong top-left.
- Breadcrumbs are acceptable on desktop for deep hierarchies.
- Back labels should describe the destination.

## Breadcrumbs

- Use breadcrumbs when navigation goes deeper than two levels.
- The current page should not be a link.
- Use a chevron separator.
- Place breadcrumbs below nav and above the page heading.

## Tabs

- Use tabs only for related views in the same context.
- Keep the count to 5-6 max.
- Never nest tabs inside tabs.
- Tabs should use nouns, not verbs.
