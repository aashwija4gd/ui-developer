# Components

## Buttons

- Use one primary action per screen or section.
- Secondary, ghost, danger, and link buttons should each have clear roles.
- Keep button labels specific and action-oriented.
- Icon-only buttons need accessible labels.

## Forms

- Labels must be visible above inputs.
- Placeholders are hints, not labels.
- Error messages should explain the fix.
- Required fields should be clearly marked.
- Group related fields together.

## Tables

- Right-align numbers and left-align text.
- Keep headers aligned with their data.
- Sortable columns need indicators.
- Truncate long text with ellipsis when needed.
- Keep actions on the right and checkboxes on the left.

## Badges and status indicators

- Use both color and label for status.
- Keep statuses consistent across the product.
- Limit the number of distinct states.

## Empty states

- Include illustration or icon, clear heading, explanation, and CTA.

## Loading states

- Use skeletons for initial loads.
- Keep layout stable while loading.
- Avoid context-free spinners for long waits.
