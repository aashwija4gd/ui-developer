# Color

## Semantic system

- Use semantic color roles consistently: primary, success, warning, error, info, neutral.
- Never use red or green outside their semantic meanings.
- Do not rely on color alone to communicate meaning.

## Contrast

- Keep text and component contrast accessible.
- Body text should meet at least WCAG AA contrast requirements.
- Check icons and UI components for legibility too.

## Dark mode

- Dark mode needs its own palette.
- Do not simply invert colors.
- Use surfaces, borders, and reduced saturation to maintain depth.

## Data visualisation

- Use labels, shapes, or patterns alongside color.
- Do not assume users can distinguish red/green combinations.
