# Interaction

## Feedback

- Every action needs visible feedback.
- Clicks should show immediate active feedback.
- Form submission should show loading, then success or error.
- Background processes should show progress and completion.

## Destructive actions

- Require confirmation for irreversible actions.
- Confirm dialogs must explain what changes, why, and whether it can be undone.
- The cancel action should be clear and easy to find.

## Hover states

- Every interactive element needs a hover state on desktop.
- Hover should be subtle and should not imply the action is already completed.
- Never rely on hover alone for mobile.

## Focus states

- Never remove focus outlines without a replacement.
- Focus rings must be visible and consistent.
- Focus order must follow visual reading order.

## Error handling

- Validate inline where possible.
- Show form-level summaries when multiple errors exist.
- Network errors should explain what happened and offer retry.
- Permission and 404 errors should give a clear next step.
